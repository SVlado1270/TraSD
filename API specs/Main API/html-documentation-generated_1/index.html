post /trasd/api/v1/upload/image
(processImage)
An user can upload an image (png/jpg/etc.) and it will be processed by the backend using our Traffic Sign Classifier. This route will return a list with the ids of the recognized signs (We use ids so we can map easily these things to our ML model) and an image with the identified signs in red squares and their name.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • image/jpg

Request body

body Object (optional)
Body Parameter —

Return type

inline_response_200

Example data

Content-Type: application/json
{
  "image" : "image",
  "signs" : [ 0, 0 ]
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

post /trasd/api/v1/upload/video
(processVideo)
An user can upload a video (mp4) and it will be processed by the backend. Long story short we will detect and classify the signs in the footage. Whenever a sign is recognised and classified we draw a red square around it and save a snapshot (a frame from the video). We will return a list with every frame that contains a classified sign and its id.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • video/mp4

Request body

body Object (optional)
Body Parameter —

Return type

array[inline_response_200_1]

Example data

Content-Type: application/json
[ {
  "image" : "image",
  "signs" : 0
}, {
  "image" : "image",
  "signs" : 0
} ]
